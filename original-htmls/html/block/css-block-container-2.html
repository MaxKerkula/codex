Position and Size of a Block Container

When the position of a block container is not specified, it is positioned after its previous block. The width and height will change based on values optionally specified and the contents included in the area. This sample demonstrates various sizing properties for block containers including logical width, logical height, minimum and maximum constraints, and overflow behavior using Antenna House Formatter's -ah-logical-* properties.
Default Block Container

This is a block area before the block container whose position isn't specified.

This is the contents of the block container whose position isn't specified. It's arranged following the previous block because the position isn't specified. Because the width and the height aren't specified, the size reflects the contents included in the area.
Width-Only Specification

This block container has -ah-logical-width: 50%:

This is the contents of the block container whose position isn't specified. It's arranged following the previous block because the position isn't specified. The width is set to 50% of the containing block, while height adjusts to content.
Width and Height Specification

This block container has -ah-logical-width: 50% and -ah-logical-height: 20%:

This is the contents of the block container with both width and height specified. The container is constrained to 50% width and 20% height of its containing block.
Fixed Height Specification

This block container has -ah-logical-width: 50% and -ah-logical-height: 30mm:

This is the contents of the block container with a fixed height of 30mm. The width is percentage-based while the height uses absolute units (millimeters).
Minimum Size Constraints

This block container has minimum constraints with -ah-min-logical-width: 30em and -ah-min-logical-height: 2em:

This is the contents of the block container with minimum size constraints. The container will not shrink below 30em width or 2em height, ensuring content remains readable.
The minimum values are specified the same as the block container just before this area.
Maximum Size Constraints

This block container has maximum constraints with -ah-max-logical-width: 30em and -ah-max-logical-height: 2em:

This is the contents of the block container whose position isn't specified. By specifying the width as -ah-max-logical-width: 30em and the height as -ah-max-logical-height: 2em, maximum values of the width and height can be specified.
The maximum values are specified the same as the block container just before this area.
