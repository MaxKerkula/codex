Controlling line breaking at abbreviations
The maximum number of characters considered to be an abbreviation can be specified using -ah-abbreviation-character-count. Line breaks can normally occur within “km/h”, “w/o”, etc. as a line break is allowed immediately after the “/”, but it can be avoided by setting -ah-abbreviation-character-count. The number of characters of the word right after “/” is less than or equal the specified number with -ah-abbreviation-character-count, the word is considered as the abbreviation and the line does not break. The default -ah-abbreviation-character-count value of ‘auto’ uses the ‘abbreviation-character-count’ value from the Option Setting File, which defaults to 3.
-ah-abbreviation-character-count: auto;
sample sample sample sample sample sample sample sample km/hr sample
-ah-abbreviation-character-count: 2;
sample sample sample sample sample sample sample sample km/hr sample
-ah-abbreviation-character-count: 1;
sample sample sample sample sample sample sample sample km/hr sample
